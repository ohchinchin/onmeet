# AI-Debate Studio プロジェクト定義書

## 1. 要件定義 (Requirements Definition)

### 1.1. 概要
複数のAIエージェントが、それぞれの専門職（ロール）の視点から特定のトピックについて議論し、司会者が進行・総括を行うWebアプリケーション。

### 1.2. 主要機能
- **動的モデル取得**: 起動時に OpenRouter API から無料モデル（Price 0）を動的に取得し、ユーザーが選択可能にする。
- **エージェント設定**:
  - 最大5名 ＋ 司会者1名。
  - 30種類の知的労働職（名前・性格付け済み）から選択。
  - 各エージェントに個別のOpenRouter Freeモデルを割り当て可能。
  - 「ランダム設定」ボタンでエージェントとモデルを自動組み合わせ。
- **議論エンジン**:
  - ターン制進行。
  - 司会者（Moderator）による進行促進（キーポイントでの介入）。
  - 設定ターン数終了後、司会者による総括。
  - 人間らしい入力遅延（タイピングエフェクト）のシミュレーション。
- **出力機能**:
  - 議論内容・総括を Markdown 形式でダウンロード。
- **インフラ・運用**:
  - Git によるバージョン管理。
  - Vercel へのデプロイ（環境変数は Vercel 側で管理、`.env` は Git 除外）。

## 2. スキーマ定義 (Schema Definition)

### 2.1. エージェント (Agent)
```typescript
interface Agent {
  id: string;
  name: string;
  role: string;        // 30種類の職種
  personality: string; // 性格・口調
  modelId: string;     // OpenRouter モデルID (例: "google/gemini-2.0-flash-exp:free")
  isModerator: boolean;
}
```

### 2.2. メッセージ (Message)
```typescript
interface Message {
  id: string;
  senderName: string;
  role: string;
  content: string;
  timestamp: string;
  isModerator: boolean;
  typingDelay?: number; // 表示までの待機時間(ms)
}
```

### 2.3. 議論セッション (DebateSession)
```typescript
interface DebateSession {
  topic: string;
  participants: Agent[];
  moderator: Agent;
  maxTurns: number;
  currentTurn: number;
  messages: Message[];
  status: 'setting' | 'running' | 'completed';
}
```
