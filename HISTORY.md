# Project History: AI-Debate Studio (onmeet)

## 2026-03-02: Stability and Reliability Update (v1.1.0 - v1.2.0)

### 課題
- ユーザー環境で 400 (Bad Request), 403 (Forbidden), 429 (Too Many Requests) エラーが頻発。
- APIエラーが発生した際、AI（特に司会者）がエラーメッセージを議論の一部と誤認し、存在しない発言を捏造して総括を行う「ハルシネーション」が発生。
- セキュリティ上の懸念（.env ファイルが Git 管理下に入りかけるリスク）。

### 実施した対策

#### 1. API通信の堅牢化 (v1.1.0)
- **OpenRouter仕様への完全準拠**: `HTTP-Referer` および `X-Title` ヘッダーを追加し、403エラーを解消。
- **タイムアウトの延長**: APIのレスポンス待機時間を 30s から 45s に延長。
- **エラーハンドリングの強化**: バックエンド (`api/chat.js`) でエラー原因（401, 429, 400等）に応じた日本語メッセージを返すよう修正。

#### 2. セキュリティ強化
- **.gitignore の修正**: `.env` を明示的に追加。
- **機密情報の保護**: APIキーが未設定の場合、サーバーをクラッシュさせずに適切なエラーレスポンスを返すようガードを実装。

#### 3. 議論ロジックの改善 (v1.1.1)
- **429レートリミット対策**: 無料モデル利用時の制限を考慮し、429エラー発生時に3秒待機して自動リトライ（1回）する機能をフロントエンドに実装。
- **履歴（History）のクリーンアップ**: `[エラー]` で始まるシステムメッセージを、AIに送る議論履歴から除外する `skipHistory` フラグを導入。

#### 4. ハルシネーション対策 (v1.2.0 - Current)
- **捏造の厳禁**: 司会者のシステムプロンプトを大幅に強化。「履歴に存在しない意見を捏造・補完することは厳禁」と強く指示。
- **双方向のエラーフィルタリング**:
    - **バックエンド**: 送信された履歴から `Error:` や `[エラー]` を含むコンテンツを物理的に除去。
    - **フロントエンド**: AIからの回答に `Error:` 文字列が含まれていた場合、それを正常な発言として受理せず、即座に議論を安全に中断するようロジックを厳格化。
- **バージョン管理の徹底**: `package.json` と UI上のバージョン表示を同期（v1.2.0）。

### 現在の状態
- OpenRouter APIとの通信が安定。
- 万が一のエラー発生時も、AIが嘘をついて議論をまとめることがなくなり、システムとしての信頼性が向上。
- セキュリティ設定（.env除外）が完了。
